# phpStorm+Vagrant 配置xdebug

因为手动打断点 复杂还需要排查问题的位置等 debug 可以一步步排查问题

1、查看php版本是否支持xdebug

执行命令 php -m
# 如无 具体的安装及配置可谷歌 (opens new window)

2、配置方法1

不配置固定的key 每一次调试都是一个新的key

打开PhpStorm
找到File | Settings | Languages & Frameworks | PHP
选择php版本 图1
如未新增过  则新增一个 图2
    如果用的是Docker 也可选择docker的镜像
找到File | Settings | Languages & Frameworks | PHP | Servers
    给项目映射一个路径到虚拟机/容器(映射目录即可) 图3
新增一个类型 即可愉快的进行xdebug调试了





3、配置方法2

配置固定的key=phpstorm 每一次调试都是固定的 和phpstorm进行关联 具体相关修改 File | Settings | Languages & Frameworks | PHP | Debug 的配置 php + xdebug 扩展时 也需要指定固定的phpstorm
