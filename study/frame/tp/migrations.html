框架如何使用数据迁移及填充 文档为个人的使用总结 如有问题请及时指正

# 1、背景

问题1：为什么要使用呢？ 问题2：使用它们有什么好处呢？

每次上线等操作  都需要手动执行sql脚本  效率低  且容易忘记 造成线上出现问题
使用它们可以更好 更方便的操作数据库的更改 更新生产环境时 只需要执行命令就搞定

# 2、数据迁移的使用

  1. 创建迁移文件 1.1 进入项目目录执行 php think migrate:create AddUserRechargeField 1.2 需要注意的是 大驼峰命名法 1.3 会在项目目录/database/migrations/创建以当前日期为开头的文件
  2. 书写迁移的代码 2.1 创建的迁移文件中有注释 可以使用的方法 2.2 change方法是tp的大坑 直接删除掉 2.3 新建 up和down方法 2.4 填写需要写的代码
  3. 执行 3.1 执行php think migrate:run 会自动执行up方法 3.2 执行php think migrate:rollback 会自动执行down方法 4.End

# 3. 数据填充操作简介

  1. 创建填充文件 1.1 php think seed:create ChangeRechargeShopNameDefault 1.2 文件命名规则 大驼峰
  2. 写数据库的操作命令 2.1 和平常一样 数据库的增删查等
  3. 执行 3.1 php think seed:run -s ChangeRechargeShopNameDefault 3.2 如果不加 -s 指定执行的填充文件 则会全部执行一遍填充 需谨慎执行 参考网站 (opens new window)
